## 1) О проекте

`Task Control` — это локальное приложение на Python и Kivy для эффективного управления личными задачами. Проект сочетает в себе классический графический интерфейс и современный ИИ-чат для управления задачами голосом или текстом на естественном языке.

### Ключевые возможности:
- **Полный CRUD:** Создание, редактирование, удаление и просмотр задач.
- **Организация:** Категории (работа, личное, покупки, здоровье, другое), приоритеты (низкий, средний, высокий) и гибкая периодичность (одноразовая, ежедневно, еженедельно, ежемесячно).
- **ИИ-Ассистент:** Интеграция с Google Gemini для понимания команд на русском языке.
- **Архивация:** Автоматическое перемещение выполненных задач в архив.
- **Статистика:** Наглядный обзор состояния ваших дел.
- **Локальное хранение:** Все данные хранятся в локальной базе данных SQLite.
- **Уведомления (Android):** Системные напоминания о предстоящих задачах через AlarmManager.

## 2) Технологический стек

### Основное
- **Runtime:** Python 3.11+ (текущее окружение: 3.12)
- **GUI Framework:** [Kivy 2.3.1](https://kivy.org/)
- **Database:** SQLite (стандартная библиотека `sqlite3`)
- **Graphics:** Pillow 12.2.0 (обработка иконок)

### ИИ-интеграция
- **Модель:** Google Gemini (gemini-2.5-flash-lite)
- **API:** REST API через `urllib.request` (без тяжелых внешних библиотек)
- **Логика:** Системный промт для структурированного вывода (JSON) и сопоставления с бизнес-логикой.

### Мобильная поддержка
- **Сборка:** [Buildozer](https://buildozer.readthedocs.io/) для Android (конфигурация в `buildozer.spec`).
- **Native:** Поддержка системных Date/Time Picker для Android через `pyjnius`.
- **Уведомления:** AlarmManager для планирования напоминаний, NotificationCompat для отображения.
- **Целевой API:** Android 34 (минимум 21).

## 4) Архитектура и взаимодействие

### Слои приложения:
1. **UI Layer (`ui/`):** Обработка ввода пользователя, анимации и отображение данных. Использует Material Design 3 подход.
2. **Service Layer (`services.py`):** "Мозг" приложения. Валидирует данные, рассчитывает статусы, управляет повторами задач и связывает ИИ с базой данных.
3. **Data Layer (`database.py`):** Прямое взаимодействие с SQLite. Реализует миграции схемы и сложные SQL-выборки (статистика, фильтрация).
4. **AI Layer (`ai_assistant.py`):** Преобразует текст пользователя в структурированные команды JSON, которые затем исполняются `TaskService`.

### Поток данных:
- Пользователь вводит текст в чат → `AIAssistant` отправляет запрос в Google Gemini API
- Gemini возвращает JSON с командой (create, update, delete, list и т.д.)
- `TaskService` валидирует и исполняет команду
- `DatabaseManager` сохраняет/обновляет данные в SQLite
- UI обновляется через callback `on_tasks_changed`

## 6) Настройка и запуск

### 1. Установка зависимостей
```bash
pip install -r requirements.txt
```

### 2. Настройка API ключа
Для работы ИИ-чата необходим ключ Google Gemini API.  
Создайте файл `config.env` в корне проекта или установите переменную окружения:
```env
GOOGLE_API_KEY=your_api_key_here
```

### 3. Запуск на ПК
```bash
python main.py
```

### 4. Сборка для Android
```bash
buildozer android debug
```

Для релиза:
```bash
buildozer android release
```

## 8) Ограничения и известные особенности

- ИИ-чат требует активного интернет-соединения.
- В текущей версии отсутствует поддержка вложений (фото/аудио) к задачам.
- Сборка под iOS требует macOS и соответствующей настройки Buildozer (в текущей версии не поддерживается).
- Уведомления работают только на Android 5.0+ (API 21+).
- На ПК уведомления не отображаются (функция заглушена).
