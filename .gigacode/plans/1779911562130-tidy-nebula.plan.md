# План внедрения Android-напоминаний (Kivy + python-for-android)

## Контекст и результаты анализа
- `main.py`: класс `TaskControlApp` уже содержит `on_start`, `build`, `_force_layout_pass` и lifecycle-точки, куда можно безопасно добавить инициализацию канала уведомлений и методы планирования/показа.
- `services.py`: в текущем коде нет отдельных `create_task`/`update_task` методов; создание/обновление идут через `save_task(...)` и `mark_task_done(...)`.
- `buildozer.spec`: уже выставлен `android.api = 34` (подходит для каналов уведомлений), AndroidX пока не включён, permissions минимальные.
- Для `NotificationCompat` в p4a/buildozer предпочтительнее подключать AndroidX через `android.enable_androidx = True` + `android.gradle_dependencies = androidx.core:core:1.9.0` (стабильнее, чем вручную `*.aar`).

## Ограничения и ключевой выбор
- Чисто Python-файл `receiver.py` не может быть автоматически системным `BroadcastReceiver` Android после полного завершения процесса приложения без регистрации Java/Kotlin receiver в manifest.
- Поэтому для требования «работает даже когда приложение закрыто» нужен нативно зарегистрированный receiver (через `android.add_src` + `android.extra_manifest_application_arguments`), который вызывает Python-обработчик.

## Рекомендуемая реализация
1. **`main.py`**
   - Добавить:
     - `_setup_notification_channel(self)` (канал `task_reminder_channel`).
     - `_show_notification(self, title, message, task_id=None)` через `NotificationCompat$Builder`.
     - `_schedule_reminder(self, task_id, title, start_datetime, reminder_type)` с `AlarmManager.setExactAndAllowWhileIdle`.
   - В `on_start`:
     - вызвать `_setup_notification_channel()` сразу,
     - затем сохранить текущий экземпляр приложения в class-level поле для доступа из receiver,
     - оставить/сохранить существующий `Clock.schedule_once(self._force_layout_pass, 0)`.
   - Добавить защиту от не-Android окружения и от кривого времени.

2. **`receiver.py` (новый)**
   - Реализовать `on_receive(context, intent)`:
     - получить extras: `task_id`, `title`, `type`;
     - сформировать текст («через час»/«начинается сейчас»);
     - через `@mainthread` вызвать у активного `TaskControlApp` метод `_show_notification(...)`.
   - Добавить fallback: если экземпляр приложения недоступен, создать уведомление напрямую из receiver (чтобы не потерять событие).

3. **`services.py`**
   - В `save_task(...)` после create/update:
     - вычислить start_datetime из `due_date` + `due_time` (`YYYY-MM-DDTHH:MM`);
     - взять `App.get_running_app()`;
     - если доступен `_schedule_reminder`, вызвать для `start` и `before`.
   - В `delete_task(...)` добавить отмену pending alarm’ов (метод `_cancel_reminders`, если добавим в `main.py`).

4. **`buildozer.spec`**
   - Обновить:
     - `android.permissions = android.permission.INTERNET,android.permission.WAKE_LOCK,android.permission.RECEIVE_BOOT_COMPLETED`
     - `android.enable_androidx = True`
     - `android.gradle_dependencies = androidx.core:core:1.9.0`
     - `android.add_src = src`
     - `android.extra_manifest_application_arguments = src/android/extra_manifest_application_arguments.xml`
   - `android.api` уже `34`, менять не требуется.

5. **Регистрация BroadcastReceiver (нативно)**
   - Подготовить Java-класс receiver в `src/...` (минимальный мост, передающий Intent в Python).
   - Добавить `<receiver ...>` + `<intent-filter><action android:name="com.taskcontrol.reminder"/></intent-filter>` в extra manifest arguments.
   - Опционально добавить `BOOT_COMPLETED` receiver для пересоздания alarm’ов после перезагрузки (если хотим полный production-сценарий).

## Что будет отдано пользователю после реализации
- Полный `main.py` (целиком).
- Новый `receiver.py` (целиком).
- `services.py` с контекстными вставками.
- `buildozer.spec` только изменённые/новые строки.
- Доп. инструкции: куда положить `src/...` и XML, если нужны ручные файлы.

## Проверка
- Локальная статическая проверка импорта/синтаксиса Python-файлов.
- Проверка, что schedule не ставится в прошлое.
- Проверка логики requestCode для двух типов уведомлений.
- Проверка совместимости флагов `PendingIntent` на API < 23 / >= 23.
- Ручной E2E на Android:
  1) создать задачу на ближайшие 70–90 минут,
  2) свернуть/закрыть приложение,
  3) получить «за час» и «в момент начала».
