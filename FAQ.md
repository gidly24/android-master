# ❓ FAQ - Часто Задаваемые Вопросы

## 🤔 Общие вопросы

### В: Почему сборка на GitHub, а не локально на Windows?
**О:** Потому что:
- ✅ GitHub имеет Linux окружение где buildozer работает стабильнее
- ✅ Не нужно устанавливать Java, Android SDK, Gradle на свой ПК
- ✅ Не нужно возиться с путями Windows (слэши и ограничения)
- ✅ Сборка идет в облаке, ты можешь работать дальше

### В: Сколько стоит GitHub Actions?
**О:** Бесплатно! 
- GitHub Actions даёт 2000 минут в месяц бесплатно (на публичных репо - неограниченно)
- Одна сборка занимает ~20-40 минут первый раз, потом 10-20 минут
- Хватит на 50-100 сборок в месяц бесплатно

### В: Можно ли использовать для Release версии?
**О:** Да! Просто добавь в workflow `buildozer android release` вместо `debug`.

## 🚀 Первая сборка

### В: Как запустить первую сборку?
**О:** 
1. Зайди на https://github.com/gidly24/android-master/actions
2. Выбери "Build Android APK"
3. "Run workflow" → "Run workflow"
4. Жди 20-40 минут

Или просто делай `git push` на `main` - автоматически запустится.

### В: Как долго ждать?
**О:** 
- Первый раз: **20-40 минут** (качает Gradle, NDK, SDK)
- Следующие разы: **10-20 минут** (всё в кэше)

### В: Где смотреть прогресс?
**О:** Actions → твоя сборка → "Build APK (Debug)" → логи в реальном времени

## 📥 Скачивание APK

### В: Где скачать готовый APK?
**О:**
1. Actions → успешная сборка (зелёный галочка)
2. Scroll в "Artifacts"
3. Клик на `taskcontrol-apk`
4. Скачается ZIP, распакуй, внутри `taskcontrol-1.0-debug.apk`

### В: Какой файл устанавливать?
**О:** `taskcontrol-1.0-debug.apk` - это Android приложение. Просто скопируй на телефон и открой.

### В: Может ли APK быть вирусом?
**О:** Нет! Это твой код, собранный GitHub Actions. Никаких третьих сторон.

## 🔴 Ошибки и проблемы

### В: Сборка упала с ошибкой "gradle not found"
**О:** Скорее всего проблема в `buildozer.spec`. Я уже исправил основное, но проверь:
- `android.gradle_dependencies` - правильные ли зависимости?
- `android.api = 34` - нужна ли такая версия?

### В: Ошибка "permission denied"
**О:** На GitHub у тебя должны быть права на запись Actions. Проверь:
- Settings → Actions → Permissions
- Должно быть "Allow all actions and reusable workflows"

### В: APK слишком большой (>100MB)
**О:** Нормально для первого раза. Включаются все ресурсы и зависимости.
Оптимизировать можно позже.

### В: "Artifact not found" или пустой APK
**О:** Значит сборка упала ДО создания APK. Смотри лог на какой строке ошибка.
Обычно там видно что не так.

## 💬 Мой код и конфиг

### В: Я изменил код, как запустить новую сборку?
**О:**
```bash
git add .
git commit -m "описание изменения"
git push origin main
```
Сборка запустится автоматически через 30 секунд.

### В: Изменил buildozer.spec, поможет ли это?
**О:** Да! После `git push` будет новая сборка с твоими изменениями.

### В: Можно ли собирать Release вместо Debug?
**О:** Да, нужно изменить в `.github/workflows/build-android.yml`:
```yaml
- run: buildozer android release
```
Но для Release нужны ключи подписания (keystore file).

## 🔧 Debugging

### В: Как понять что не так если сборка упала?
**О:** 
1. Клик на красный статус сборки
2. Scroll вниз в "Build APK (Debug)"
3. Ищи строку с `ERROR` или `failed to`
4. Google эту ошибку + "buildozer android"
5. Если не поймёшь - скопируй последние 50 строк и покажи мне

### В: Можно ли собирать несколько архитектур одновременно?
**О:** Можно, но медленнее. В `buildozer.spec`:
```
android.archs = arm64-v8a, armeabi-v7a
```
Пока оставил только `arm64-v8a` для скорости.

### В: Что если gradle зависает?
**О:** GitHub Actions имеет timeout 360 минут. Если зависает дольше - может быть проблема в `build.gradle` или зависимостях. Можно добавить таймаут или переделать конфиг.

## 📚 Дополнительное

### В: Какая версия Python используется?
**О:** Python 3.11 (в workflow указано `python-version: '3.11'`)

### В: Какие версии Android API поддерживаются?
**О:** 
- `android.api = 34` - целевая версия (Android 14)
- `android.minapi = 21` - минимальная (Android 5.0)

### В: Как добавить ещё одно приложение в сборку?
**О:** Создай новый `.github/workflows/build-something-else.yml` файл с похожей структурой.
