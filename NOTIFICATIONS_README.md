# Уведомления для Android

## Описание

Приложение теперь поддерживает нативные уведомления на Android с использованием `AlarmManager` и `BroadcastReceiver`.

### Функционал

- **Уведомления за 1 час до начала задачи** - напоминает о предстоящей задаче
- **Уведомления в точное время начала задачи** - уведомляет о начале задачи
- **Автоматическое планирование** - уведомления планируются при создании/изменении задачи
- **Отмена уведомлений** - при выполнении/удалении задачи уведомления отменяются
- **Периодические задачи** - при выполнении повторяющейся задачи уведомления перепланируются на следующий срок

## Структура файлов

```
android/
└── PythonBroadcastReceiver.java          # BroadcastReceiver для обработки уведомлений
main.py                                    # Основной файл с логикой уведомлений
services.py                                # Сервис задач с callback для уведомлений
buildozer.spec                             # Конфигурация сборки с разрешениями
```

## Как это работает

1. **Планирование уведомлений**:
   - При создании/изменении задачи вызывается `_schedule_task_reminder`
   - Создаются два уведомления: "за час" и "вовремя"
   - Используется `AlarmManager.setExactAndAllowWhileIdle()` для точного времени

2. **Обработка уведомлений**:
   - `PythonBroadcastReceiver` получает уведомление от `AlarmManager`
   - Создает нативное уведомление с помощью `NotificationCompat.Builder`
   - При клике на уведомление открывается приложение с фокусом на задачу

3. **Отмена уведомлений**:
   - При выполнении/удалении задачи вызывается `_cancel_reminders`
   - Отменяются все запланированные уведомления для задачи

## Настройка buildozer.spec

В `buildozer.spec` должны быть следующие настройки:

```ini
# Разрешения
android.permissions = android.permission.INTERNET, android.permission.WAKE_LOCK, android.permission.ALARM_SERVICE

# Java файлы
android.add_src = android
```

## Сборка приложения

1. Установите зависимости:
```bash
pip install buildozer
```

2. Инициализируйте buildozer (если еще не инициализирован):
```bash
buildozer init
```

3. Соберите APK:
```bash
buildozer android debug
```

4. Установите на устройство:
```bash
buildozer android deploy run
```

## Разрешения

Приложению требуются следующие разрешения:

- `INTERNET` - для работы AI-ассистента
- `WAKE_LOCK` - для пробуждения устройства при уведомлении
- `ALARM_SERVICE` - для использования AlarmManager

## Тестирование

### На эмуляторе

1. Создайте эмулятор с Android API 26+ (для каналов уведомлений)
2. Соберите и установите приложение
3. Создайте задачу с датой и временем в будущем
4. Подождите until уведомление (или измените время на устройстве)
5. Проверьте, что уведомление появилось

### На реальном устройстве

1. Установите приложение на устройство
2. Создайте задачу с датой и временем в будущем (например, через 5 минут)
3. Закройте приложение
4. Подождите until уведомление
5. Нажмите на уведомление, чтобы открыть приложение

## Отладка

### Логи уведомлений

В `main.py` добавлены print-сообщения для отладки:

```python
print(f"[Уведомления] Кликнули по задаче с ID: {task_id}")
```

### Проверка AlarmManager

Для проверки запланированных уведомлений можно использовать:

```bash
adb shell dumpsys alarm | grep taskcontrol
```

### Проверка разрешений

Убедитесь, что приложение имеет все необходимые разрешения:

```bash
adb shell pm list permissions -g -d org.ivn.taskcontrol
```

## Известные проблемы

1. **Уведомления не приходят**:
   - Проверьте, что устройство не в режиме "Не беспокоить"
   - Убедитесь, что приложение не в режиме экономии батареи
   - Проверьте разрешения в настройках устройства

2. **Уведомления приходят с задержкой**:
   - Android может откладывать уведомления для экономии батареи
   - Используется `setExactAndAllowWhileIdle`, но на некоторых устройствах это не работает

3. **Уведомления не удаляются**:
   - Убедитесь, что вызывается `_cancel_reminders` при удалении задачи
   - Проверьте, что `task_id` совпадает
